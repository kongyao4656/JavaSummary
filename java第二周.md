# 学习总结
## 进阶内容
1.看了并行，多线程，线程池的概念，多线程可以共用一块资源，为了避免竞争冲突导致的数据错乱，就需要用同步或者锁，用同步的方法就是使用synchronize关键字，syn关键字放在实例方法上，就代表当对象调用这个方法时，会将对象锁住，也就是只有这个对象才能调用这个方法，当这个对象调用完了这个方法，那么就可以释放锁，让别的对象调用这个方法。syn关键字也可以锁代码块，当锁代码块的时候，只有代码块中的内容是上锁的，其他的内容不上锁，可以并行执行。然后就是锁的概念，锁更加灵活一点，锁可以有公平锁，非公平锁，条件锁，悲观锁，乐观锁。在运行一个代码时候，首先上锁能够将函数锁上，最后运行完代码再释放锁。锁有条件锁，条件锁的作用是进行线程间通信，比如有的线程抽取队列当中的元素进行处理，当队列为空时负责抽取的线程需要等待，当负责增加的线程增加一个元素，就通过条件锁通知抽取的线程可以唤醒。如此实现线程间通信。
2.自己通过java的实例做了实验，知道了一些关于多线程的细节，就是当等待的线程被唤醒时，它只是被唤醒，而并没有获得锁，只有当有锁线程执行完毕，释放锁后才有可能让唤醒线程获得锁，注意这里是有可能，而不是一定，因为一个代码里面，比如有两行产生新线程的，不一定谁先运行谁后运行的，也就是说不一定谁先获得锁。
